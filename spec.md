# FinderTerm 仕様書(ドラフト)

MacのFinderウィンドウ内(下半分)に、表示中のディレクトリに追従するターミナルを埋め込んで見せるアプリ。Claude Codeをフォルダごとにすぐ起動できるようにするのが主目的。

**ステータス**: 全セクション執筆済み。最終レビュー待ち。

---

## 1. 背景と方針

- macOSのFinderは内部にビューを注入する拡張を許していない(SIPにより、旧TotalFinder方式のコード注入は不可。公式のFinder拡張はバッジ・ツールバーボタン・右クリックメニューまで)。
- そのため「本物のFinder内部」ではなく、**枠なしターミナルウィンドウをFinderウィンドウの下半分に吸着・追従させて一体化して見せる**方式を採る(ユーザー承認済み)。

## 2. 決定済み要件

| # | 項目 | 決定 |
|---|------|------|
| R1 | 実現方式 | Finderウィンドウに吸着する別ウィンドウ(SIP無効化なし) |
| R2 | レイアウト | Finderウィンドウの**下半分に重ねる**(オーバーレイ)。デフォルト高さ40%、境界ドラッグで調整可 |
| R3 | 複数ウィンドウ | **Finderウィンドウごと**に独立したペイン+シェルセッション |
| R4 | cd追従 | Finderでフォルダ移動したら、**シェルがアイドルのときだけ**自動cd。claude等の実行中は何もしない |
| R5 | セッション寿命 | Finderウィンドウを閉じるとき、実行中プロセスがあれば**確認ダイアログ**。アイドルなら黙って終了 |
| R6 | 表示トグル | グローバルホットキー(デフォルト ⌥⌘T)で**全ペインを一括**表示/非表示(アプリ全体のオン/オフスイッチ) |

## 3. セクション1: 全体アーキテクチャ

**アプリの形態**: メニューバー常駐のSwiftアプリ(Dockアイコンなし、`LSUIElement`)。仮称 **FinderTerm**。Swift Package Manager + AppKitで構築。ターミナルエミュレーションはOSSの [SwiftTerm](https://github.com/migueldeicaza/SwiftTerm) を使用(La Terminal等での採用実績あり)。

### コンポーネント構成(5つ)

1. **FinderWindowTracker**
   アクセシビリティAPI(`AXObserver`)でFinderの全ウィンドウを監視。「出現・移動・リサイズ・最小化・フォーカス・破棄・タイトル変更」をイベントで受け取る。通常のブラウザウィンドウだけをフィルタし(情報ウィンドウ・ダイアログは除外)、下流にイベントを流す。

2. **PathResolver**
   各Finderウィンドウが表示中のフォルダのPOSIXパスを解決する。AppleEvents(Finderへのスクリプティング)で `window id` 単位に問い合わせる。タイトル変更イベント(=フォルダ移動)をトリガーに再解決。「最近の項目」「AirDrop」「検索結果」などパスを持たない画面は「パスなし」として扱う。

3. **PaneController**(Finderウィンドウごとに1つ)
   枠なしのターミナルウィンドウを、対応するFinderウィンドウの下半分に重ねて配置。移動・リサイズ通知のたびに自フレームを同期し、z順序も常にFinderウィンドウの直上に維持する(他アプリのウィンドウは普通に両者を覆える)。Finder側が最小化・別Space・非表示なら一緒に隠れる。

4. **TerminalSession**(ペインごとに1つ)
   SwiftTermの `LocalProcessTerminalView` でユーザーのログインシェル(`$SHELL -l`)を起動。アイドル判定とcd注入を担当(詳細はセクション2で定義)。

5. **AppCoordinator + メニューバーUI**
   上記を束ねる。グローバルホットキーでペイン表示/非表示をトグル。メニューバーから設定(ペイン高さ、ホットキー、ログイン時起動)と終了。初回起動時に「アクセシビリティ」「オートメーション(Finder制御)」の2権限の取得を案内するオンボーディングを持つ。

### 設計上の注意点

- **子ウィンドウAPIは使えない**: macOSの `addChildWindow` は他アプリのウィンドウに対して使えないため、追従はAXイベント駆動で自前同期する。ポーリングではなくイベント通知なので体感遅延はほぼゼロにできる。
- **必要権限**: アクセシビリティ(ウィンドウ追跡)とオートメーション/AppleEvents(Finderへのパス問い合わせ)の2つ。どちらもユーザーが一度許可すれば済む。SIP無効化・コード注入は一切不要。

## 4. セクション2: Finder追従とcd注入の仕組み

### 4.1 ウィンドウの同定

- Finderの各ウィンドウは `AXUIElement` として取得し、`_AXUIElementGetWindow()`(私用APIだがユーティリティ系アプリで広く使われ安定)で **CGWindowID** を得る。
- このCGWindowIDは、FinderのAppleScriptにおける `window id` と一致する。これにより「AXで追跡しているウィンドウ」と「スクリプティングでパスを問い合わせるウィンドウ」を確実に対応付けられる。z順序制御(`order(.above, relativeTo:)`)にも同じIDを使う。

### 4.2 位置追従

- **イベント駆動が基本**: `AXObserver` で移動・リサイズ・最小化/復元・破棄・タイトル変更・フォーカス変更の通知を受け、そのたびにFinderウィンドウの `AXPosition`/`AXSize` を読んでペインのフレームを `(x, y + h·(1−r), w, h·r)`(r=ペイン高さ比率、デフォルト0.4)に設定する。
- **ドラッグ中の平滑化**: ウィンドウドラッグ中はAX通知の間隔が粗くなることがあるため、最初の移動通知からマウスアップまでの間だけ60Hzのポーリングで補間する。ドラッグ終了後はポーリングを止め、イベント駆動に戻る。
- **Space / Mission Control**: `NSWorkspace` のSpace切替通知で、対応Finderウィンドウが現在のSpaceの画面上にあるか(`CGWindowListCopyWindowInfo`)を再評価し、なければペインを隠す。
- **v1の制限**: フルスクリーンのFinderには追従しない(別アプリのフルスクリーンSpaceには枠なしウィンドウを重ねられないため、ペイン非表示)。

### 4.3 パス解決

- タイトル変更通知(=フォルダ移動)とウィンドウ出現時に、AppleEventsで `tell application "Finder" to get POSIX path of (target of window id N as alias)` を実行して現在フォルダを得る。
- AppleEventsはFinderがビジーだと待たされうるので、バックグラウンドキュー+タイムアウト(2秒)で実行し、UIをブロックしない。
- **パスを持たない画面**(最近の項目・AirDrop・検索結果・ゴミ箱等)ではエラーが返るので、その間は最後の有効パスを保持し、cd注入は行わない。
- 連続ナビゲーション(キーボードでフォルダを次々開く等)に備え、解決とcd注入は300msデバウンスする。

### 4.4 アイドル判定とcd注入

- シェルはSwiftTermの `LocalProcessTerminalView` から `$SHELL -l` で起動(ptyのマスタFDとシェルPIDを保持)。
- **アイドル判定**: `tcgetpgrp(masterFD)` が返すフォアグラウンドプロセスグループがシェル自身のものなら「プロンプト待ち」とみなす。claude・vim・ビルド等が前面で動いていれば異なるpgidになるので注入しない。
- **無駄打ち防止**: 注入前に `proc_pidinfo(PROC_PIDVNODEPATHINFO)` でシェルの実際のcwdを取得し、移動先と同じならスキップ。
- **注入方法**: ptyに `Ctrl-U`(入力中の行をクリア)→ `cd '<エスケープ済みパス>'` + 改行 を書き込む。コマンドは画面にエコーされる(Terminal.appの「新規タブでcd」と同じ見え方)。
  - 注意: ユーザーが打ちかけて実行していないコマンドラインはCtrl-Uで消える。追従の確実性を優先した割り切り(仕様として明記)。
  - パス中のシングルクォートは `'\''` 形式でエスケープする。
- **実行終了後の再同期**: claude等が終了してフォアグラウンドがシェルに戻ったことを検知(1秒間隔の軽量ポーリング)したら、Finderの現在フォルダへ一度だけ再同期する。

### 4.5 ペインUI

- ペイン上端は細いドラッグハンドル(境界バー)。ドラッグで高さ比率rを変更でき、値はグローバル設定として永続化(ウィンドウ個別ではなく全ペイン共通)。
- 外観はシステムのライト/ダークに追従。設定はフォント名・サイズ程度の最小限にとどめる。

## 5. セクション3: ライフサイクル・エラー処理・テスト・配布

### 5.1 セッションのライフサイクル

- **Finderウィンドウが閉じられたとき**(AX破棄通知):
  - セッションがアイドル → シェルを終了(ptyクローズによるSIGHUP)し、ペインを破棄。
  - 実行中プロセスあり → 確認ダイアログ「実行中のプロセスがあります。終了しますか?」
    - **終了**: シェルごと終了。
    - **セッションを残す**: ペインは通常の独立フローティングウィンドウに切り替わって残る(Finderウィンドウはもう無いので吸着解除)。以後は普通のターミナルウィンドウとして使え、閉じれば終了。
- **アプリ終了時**: 実行中セッションが1つでもあれば、件数を示して一括確認。
- **Finderの再起動**(`killall Finder` 等): 旧ウィンドウとの対応は復元できないため「ウィンドウが閉じられた」と同じ扱い(実行中セッションは独立ウィンドウ化して守る)。Finder復帰後の新しいウィンドウには新規ペインを割り当てる。
- **スリープ復帰・ディスプレイ構成変更**: 全ペインのフレームを再同期する。

### 5.2 エラー処理

| 状況 | 挙動 |
|------|------|
| アクセシビリティ権限なし(初回) | オンボーディングウィンドウで2権限の設定手順を案内。許可されるまでペインは出さない |
| 権限が実行中に剥奪された | 30秒間隔の`AXIsProcessTrusted`チェックで検知。全ペインを隠し、メニューバーアイコンを警告状態にして再設定を案内 |
| オートメーション(AppleEvents)拒否 | 位置追従は継続するが、cd追従だけ停止。メニューバーで警告 |
| AppleScriptタイムアウト(2秒) | 最後の有効パスを保持し、次のイベントで再試行 |
| シェルが終了(exit・クラッシュ) | ペインに「セッション終了 — クリックで再起動」を表示。クリックでFinderの現在フォルダを起点に新シェルを起動 |

### 5.3 テスト

- **ユニットテスト(XCTest)**: 純粋ロジックに切り出してテストする。
  - フレーム計算(Finderフレーム+比率r → ペインフレーム)
  - パスのシングルクォートエスケープ
  - cd注入バイト列の組み立て(Ctrl-U + cd + 改行)
  - アイドル判定の判断ロジック(pgid比較・cwd一致スキップ・デバウンス)※システムコール部分はプロトコルで抽象化しモックを注入
- **統合テスト**: AX追跡とAppleEventsは自動化が難しいため、手動チェックリストを `TESTING.md` に置く(ウィンドウを開く/動かす/リサイズ/フォルダ移動/最小化/Space切替/閉じる、claude実行中のcd抑制、確認ダイアログ)。

### 5.4 ビルドと配布

- **対象OS**: macOS 13 (Ventura) 以降。
- **ビルド**: Swift Package Manager の実行可能ターゲット+`.app` バンドル組み立てスクリプト(`Info.plist` に `LSUIElement=true` を設定)。Xcodeプロジェクトは持たない。
- **署名**: 個人利用のためad-hoc署名。バンドルIDを固定し、再ビルドで権限の再許可が要求されにくいようにする。
- **配布しない前提**: AX権限が必要なためサンドボックス化不可=App Store不可。当面は自分用ローカルビルドのみ。公開するなら将来Developer ID署名+公証を検討。

## 6. 成功基準

1. Finderウィンドウをドラッグ・リサイズしてもペインが遅延なく追従し、「内蔵ペイン」に見える。
2. Finderでフォルダを移動すると、アイドル中のシェルが自動でそのフォルダにcdされる。
3. ペインで `claude` を起動して作業中、Finderでフォルダを移動してもセッションが乱されない。
4. Finderウィンドウを2枚以上開くと、それぞれに独立したペイン+セッションが付く。
5. claude実行中のウィンドウを閉じようとすると確認が出て、「残す」を選べばセッションが独立ウィンドウとして生き残る。
6. ⌥⌘T でペインの表示/非表示が切り替わる。
